# Novelty Review, July 2026

The literature now contains mature network digital twin surveys, data-driven performance models, closed-loop SLA management, AI-driven network twins, self-adaptive twin synchronization, causal network optimization, and early counterfactual validation. A generic network twin or traffic predictor is therefore insufficient as a primary contribution.

CausalNetTwin is positioned around the combination of five controls that are not jointly provided in the reviewed work:

1. A causal world model for mixed AI, storage, and data-engineering traffic in multi-tenant data-center fabrics.
2. Action-specific counterfactual evaluation for routing, queue, ECN, and rate-control interventions.
3. A hard safety gate across tail latency, packet loss, fairness, route churn, and SLO violations.
4. Post-deployment outcome verification against the counterfactual prediction.
5. Automatic rollback with a persistent action audit trail.

The closest 2026 work, CiiNet, applies causal inference to network optimization, while recent trustworthy NDT work validates directional what-if predictions for cloud-edge resource scaling. CausalNetTwin narrows the research problem to safe agentic remediation in AI data-center fabrics and evaluates harmful-action prevention and rollback as first-class outcomes.
